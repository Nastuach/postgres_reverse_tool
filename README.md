# 🐘 PostgreSQL Reverse Engineering Tool

Инструмент для автоматического анализа структуры базы данных PostgreSQL и выявления бизнес-правил.
Приложение позволяет автоматически анализировать структуру базы данных PostgreSQL, извлекать метаданные из системных каталогов pg_catalog и выявлять бизнес-правила, реализованные через триггеры, функции и ограничения.

## ✨ Возможности

| Функция | Описание |
|---------|----------|
| 🔍 Анализ таблиц | Извлечение всей информации о таблицах схемы public |
| 📝 Анализ колонок | Типы данных, nullable, default значения, вычисляемые поля |
| 🔗 Ограничения | PK, FK, UNIQUE, CHECK ограничения с определениями |
| 📇 Индексы | Все индексы с типами, колонками и условиями |
| ⚡ Триггеры | Триггеры с исходным кодом функций |
| 🔧 Функции | Хранимые функции и процедуры PL/pgSQL |
| 📜 Бизнес-правила | Автоматическое выявление из кода триггеров и функций |
| 📊 ER-диаграмма | Визуальная схема связей между таблицами |
| 📄 Экспорт в PDF | Генерация отчёта в формате PDF |
| 📊 Экспорт в Excel | Генерация отчёта в формате XLSX |
| 💬 Комментарии | Извлечение комментариев ко всем объектам БД |

## 🚀 Быстрый старт

### 1. Установка Python

Убедитесь, что установлен Python 3.8 или выше:

```bash
python --version
```

### 2. Клонирование репозитория

```bash
git clone https://github.com/ВАШ_НИК/postgres_reverse_tool.git
cd postgres_reverse_tool
```

### 3. Создание виртуального окружения

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**Linux/Mac:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### 4. Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 5. Установка Graphviz

**Windows:**
1. Скачайте установщик с [graphviz.org](https://graphviz.org/download/)
2. Установите и добавьте в PATH (отметьте галочку при установке)
3. Перезапустите терминал

**Linux:**
```bash
sudo apt-get install graphviz
```

**Mac:**
```bash
brew install graphviz
```

### 6. Запуск приложения

```bash
python main.py
```

## 🧪 Тестирование на тестовой базе данных

Для проверки работы приложения рекомендуется использовать тестовую базу данных, которая содержит все типы объектов PostgreSQL.

### Шаг 1: Создание тестовой БД

```bash
# Подключитесь к PostgreSQL
psql -U postgres

# Создайте базу данных
CREATE DATABASE test;

# Подключитесь к новой БД
\c test

# Выполните SQL-скрипт
\i create_test_db.sql

# Или из командной строки
psql -U postgres -d test -f create_test_db.sql
```

### Шаг 2: Запуск приложения

```bash
python main.py
```

### Шаг 3: Подключение к тестовой БД

Введите параметры подключения:
- **Хост:** `localhost`
- **Порт:** `5432`
- **База данных:** `test`
- **Пользователь:** `postgres`
- **Пароль:** `ваш_пароль`

### Шаг 4: Проверка результатов

После анализа вы увидите:

| Объект | Количество |
|--------|------------|
| 📊 Таблиц | 8 |
| 📝 Колонок | ~50 |
| 🔗 Связей FK | 7 |
| 🔑 Первичных ключей | 8 |
| ✅ CHECK-правил | 6 |
| ⚡ Триггеров | 2 |
| 🔧 Функций | 3 |
| 📇 Индексов | 5 |

### 📁 Что создаёт тестовый скрипт

**Таблицы:**
- `users` — пользователи системы
- `roles` — роли пользователей
- `user_roles` — связь пользователей и ролей (M:M)
- `categories` — категории товаров (иерархия)
- `products` — каталог товаров
- `orders` — заказы
- `order_items` — элементы заказов
- `audit_log` — журнал аудита

**Ограничения:**
- PRIMARY KEY на всех таблицах
- FOREIGN KEY между связанными таблицами
- CHECK ограничения (email, цена, количество, статус)
- UNIQUE на username, email, sku

**Триггеры:**
- `trg_products_updated_at` — автообновление timestamp
- `trg_audit_orders` — аудит изменений заказов

**Функции:**
- `update_updated_at_column()` — обновление временной метки
- `calculate_order_total()` — подсчёт суммы заказа
- `check_product_availability()` — проверка наличия товара

**Индексы:**
- На email, category_id, user_id, status, order_id
